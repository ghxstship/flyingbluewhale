# Internal UI/UX Audit — ATLVS

Read-only codebase audit conducted 2026-06-11 against the seven evaluation dimensions in the audit brief. No application code was modified. Baseline: **980 `page.tsx` routes** — (platform) 634 · (portal) 125 · (mobile) 74 · (marketing) 83 · (personal) 25 · (auth) 13. Phase 2 of 4.

Each finding: **ID · Severity · Antipattern · Files · One-line remediation.** Severity counts: 2 Critical · 13 High · 21 Medium · 12 Low.

---

## 7. Heuristic Summary (Nielsen Mapping)

- **Visibility of system state** — weakest area: silent 100-row truncation (SC-2/SC-3), no pending state on ad-hoc forms (FE-8), misleading offline banner (SC-4), shell-generic skeletons (SC-6).
- **Consistency & standards** — strong primitives undermined by 144 shadow tone maps (CN-1/2), three badge idioms (CN-7), orphan primitives (CN-4/5).
- **Error prevention** — cents trap (FE-3), no dirty guards by default (FE-5), native confirms (CN-9).
- **Recognition over recall** — orphaned flagship surfaces (NV-1, NV-10), palette blind spot (NV-2), 14% breadcrumb adoption (NV-9).
- **User control / undo** — no undo anywhere despite soft-delete schema (FE-6); escape routes good (DeleteForm, error boundaries with reset).
- **Flexibility & efficiency** — bulk ops built but unwired (FE-2), no entity search (NV-5), saved views + ⌘K recents are genuinely good.

## Done Well (Keep and Propagate)

1. **FormShell** is a model form surface: useActionState pending → disabled loading button (double-submit structurally prevented in 237 forms), **submitted-values echo** after validation failure, LiveRegion announcements, full dirty-guard machinery (adoption is the gap, not construction).
2. **DataTable/DataTableInteractive**: one server wrapper over one client engine across 211 usages; @tanstack/react-virtual by default; saved views, CSV, density, debounced view persistence, filtered-empty with one-click clear; `aria-sort`, real button headers, per-row checkbox labels.
3. **Radix end-to-end primitives** + LiveRegion actually mounted and consumed (FormShell, all three scanners, Calendar/Timeline/Kanban); global `prefers-reduced-motion` kill switch; deliberate AA contrast remediation with annotated tokens and separate `--p-accent-cta` pairs (including the hard COMPVSS-amber case).
4. **Sidebar is Linear-class**: collapsible groups with active force-open, pinned favorites, inline `/` search, drag-resize, ⌘B, role filtering, per-user persistence. Zero stub destinations remain (`PageStub` count: 0).
5. **The offline queue is genuinely good where it exists**: IndexedDB FIFO, background-sync + iOS drain, capture-time stamping, 4xx-terminal/5xx-retry semantics, "Queued — will sync" toasts with pending counts. The gap is coverage and honesty, not architecture.
6. **Portal authorization leaks nothing**: slug resolution at the layout boundary → `notFound()` for unauthorized and nonexistent portals alike (e2e-enforced).
7. **Streaming exemplars exist** (projects + invoices pages: React.cache + dual Suspense islands with sized fallbacks and aria-busy) — the pattern just needs propagation.
8. **Zero raw Tailwind palette classes in src/app** — all color flows through `--p-*` variables; class-level theming discipline is excellent.
9. **API-layer 403s are explicit and auditable** (`assertCapability` envelopes naming persona/role/capability) — the page layer should match them (SC-5).
10. **Marketplace module is the template**: centralized STATUS_TONE in lib, imported by ~30 surfaces — exactly the pattern CN-1 asks the other modules to copy.
