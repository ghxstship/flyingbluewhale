## The XPMS v08 Backbone

Every event in this package is structured as a v08 budget row matrix. The same six axes carry the work, regardless of tier scale.

### Axis 1 — Phase (the 8 Gates)

| #   | Phase           | Absorbs                       | Objective Exit Gate                             |
| --- | --------------- | ----------------------------- | ----------------------------------------------- |
| 1   | **Discovery**   | (pre-design / pursuit)        | Brief approved, go decision made                |
| 2   | **Design**      | Concept, Develop, Engineering | Design package approved, scope locked           |
| 3   | **Advance**     | Pre-Production                | Contracts & POs issued, budget baselined        |
| 4   | **Procurement** | (long-lead purchasing)        | PO issued / deposit paid (committed cost)       |
| 5   | **Build**       | Fabrication, Construction     | Fab/construction complete, QC passed            |
| 6   | **Install**     | Logistics, Installation       | Installed, commissioned, punch closed, accepted |
| 7   | **Operate**     | Show, Strike                  | Operating acceptance / struck                   |
| 8   | **Close**       | Wrap                          | Reconciled, final cost report filed             |

> **Rule.** A phase is justified only if it isolates a metric you'd make a different decision on. Fee and Contingency are NOT phases — they are Line Types. Scope growth is a Change Order, not contingency consumption.

### Axis 5 — XYZ Cost Behaviour

- **X — Constant** (fixed; doesn't move with scale or schedule)
- **Y — Variable** (scales with attendees, units, or labor draw)
- **Z — Timeline / Phase** (depends on phase timing — overtime, expedite freight, premium lead-times)

### Axis 6 — Line Type

- **Scope** — real work. Phase curve sums Scope only.
- **Fee** — producer compensation. Collected on draw schedule, not absorbed into Scope rows.
- **Contingency** — reserve for in-scope variance. 5–10% typical floor.
- **Allowance** — placeholder for not-yet-specified scope.
- **Markup** — pass-through margin on third-party costs.

## Why the Live-Event Draw Variant Applies Here

The XPMS v08 default draw is **50 % Mobilization (Discovery / contract) · 30 % Progress (Build start) · 20 % Final (Close / acceptance)** — built for projects with a real, multi-week Build phase.

Casa Miami's per-event programming does not have that shape. The Casa is **already built** (the Design Concept #1 build-out is a separate engagement). Each event's Phase 5 Build line is a 2–4 day shop call, not a multi-week construction push. We therefore use the documented v08 live-event variant:

| Draw    | Trigger                                                               |       % |
| ------- | --------------------------------------------------------------------- | ------: |
| Deposit | On contract execution (Phase 1 Discovery → Phase 2 Design transition) | **60%** |
| Balance | Due five (5) business days prior to venue load-in (Phase 6 Install)   | **40%** |

This is the same 60/40 split that has historically run all live-event work in this ecosystem — but it is **now scoped as a named variant, not a global default**. The Casa _build-out_ proposal (Design Concept #1, not in this package) would default to 50/30/20.

## Casa Miami House Standards (Cross-Cutting)

The following are inherited by every event regardless of tier. They reflect the always-on infrastructure delivered by the Design Concept #1 build-out (separate engagement, not re-priced per event).

### House Infrastructure (Casa Capex — Not Event-Billable)

- **House PA** — L-Acoustics A10 hangs L+R + KS21 subs, optimised for the 250-cap Main Stage Room
- **House lighting** — 24× Astera Titan tubes + 12× Aputure 600x + 6× Robe T1 profiles on truss grid · GrandMA3 console
- **House video** — 4K broadcast switcher · 6× Sony FX6 + 2× FR7 PTZ · Podcast Studio with 4× Sennheiser MKH 416 booms
- **House power** — 200A 3-phase service · isolated AV ground · UPS-backed broadcast
- **House data** — 1 Gbps symmetric fiber + redundant LTE failover
- **House CO** — Posted occupancy load: **485 indoor · 120 patio · 605 max combined**

### House Safety Baseline (every event, every tier)

- 1× Florida-licensed, Fire Marshal-trained Event Safety Director
- 2× AED units (lobby + green room), full first-aid kit per zone, posted evac map at every door
- Sprinklered throughout per CO; no open flame or pyro without separate City of Miami special-event permit
- Standing relationships: **Star Medical** (BLS standby), **Off Duty Officers Miami** (sworn MPD security)

### House Sustainability Baseline

- Zero single-use plastic on bar (compostable or glass only)
- Front-of-house composting + recycling sort (Trashie Miami)
- Recycled stock for all print; rentable décor preferred over single-use builds

### Insurance Floor

- $5M Comm GL + $5M Liquor Liability + $5M Auto + Workers Comp statutory
- Spotify LATAM, building owner, and venue manager named additional insured on all certs
- Talent inducement riders matched per offer letter
