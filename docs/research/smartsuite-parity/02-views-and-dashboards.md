# SmartSuite Views + Dashboards — Parity Gap Analysis

> Scope: SmartSuite's "Views" collection (10 view types, ~62 documented articles) and Dashboards (widgets, charts, sharing, drill-in). Compared against FLYTEHAUS Technologies (`flyingbluewhale`) — Next.js 16 / Supabase / recharts platform with three shells (ATLVS, GVTEWAY, COMPVSS).
>
> Companion doc to `01-fields-and-records.md`. Source-of-truth on the FLYTEHAUS side: `src/components/DataTable.tsx`, `src/components/DataTableInteractive.tsx`, `src/components/charts/*`, `src/app/(platform)/console/dashboards/page.tsx`, `src/lib/hooks/useUserPreferences.ts`.

## 1. Executive summary

The single biggest finding: **FLYTEHAUS has one excellent view (Grid) and zero second-class views**. SmartSuite's competitive moat in this collection is not any one view — it's the polymorphism. The same record set rendered nine different ways, each with the same filter/sort/group/spotlight toolbar, and a tenth Dashboard surface that composes them. That is what we lack.

- **P0 — Kanban view is missing entirely.** Production ops live in Kanban (lanes by status: `not_started → in_progress → blocked → done`). We have `@dnd-kit` already wired in `DataTableInteractive` and `SortableList`; the primitive cost is one `<KanbanBoard>` component. ([SmartSuite Kanban][ss-kanban])
- **P0 — Calendar view is single-instance.** `src/app/(platform)/console/schedule/ScheduleCalendar.tsx` is read-only month/week, schedule-module-specific, no drag-to-reschedule, no day/agenda mode. SmartSuite supports day/week/month/year/list and overlays up to 5 calendars. ([SmartSuite Calendar][ss-calendar])
- **P0 — Gantt + Timeline are missing.** No Gantt component, no timeline. Production scheduling, look-ahead, and the 8-Phase Lifecycle (`feedback_8_phase_lifecycle.md`) all want a dependency-aware Gantt. ([SmartSuite Gantt][ss-gantt])
- **P0 — No shareable saved views.** `DataTableInteractive` persists per-user view state in `user_preferences.table_views[tableId]` — but there is no concept of a _named, shared_ view a team can pin. SmartSuite's Saved View panel (private vs public, locked, foldered) is the most-used feature in the entire collection. ([SmartSuite Saved Views][ss-saved-view])
- **P0 — No conditional formatting / Spotlight.** SmartSuite's Spotlight colors rows or cells by rule (status, formula, threshold). We have `StatusBadge` per cell but nothing that paints the whole row red when an RFI is overdue. ([SmartSuite Spotlight][ss-spotlight])
- **P1 — Map view is bespoke per-page.** `LiveDispatchMap` and a `MapShell` exist for dispatch only. Any address-bearing record (venues, vendors, accommodations, incidents) should be mappable from its list view without writing a new component.
- **P1 — Dashboards page is hardcoded.** `src/app/(platform)/console/dashboards/page.tsx` is one fixed Portfolio KPI grid. SmartSuite gives the user a widget palette + drag layout. Even a simple "pick 4 of these 12 KPI cards" config beats the current static grid.
- **P1 — Chart View as a polymorphism is missing.** `recharts` is in deps; `ReportsCharts.tsx` and `CarbonCharts.tsx` hand-build them per page. There is no generic "render this query as a bar/pie/line chart" component an operator can spin up against `tickets` or `invoices` without editing code.
- **P1 — Form View is not public.** `console/forms` is a CRUD list of `form_defs` rows; nothing here matches SmartSuite's public-shareable / embeddable form with conditional logic and prefill. ([SmartSuite Form][ss-form])
- **P2 — Grid lacks: column totals/summary footer, Excel-like fill handle, bulk paste, right-click context menu, undo/redo on edits.** All present in SmartSuite Grid; partially compensated by our DataTable's bulk actions + CSV export. ([Column Totals][ss-totals])

## 4. Dashboards & widgets

SmartSuite's Dashboard is "a special View that pulls data from any table." Widget categories per [Dashboard creation][ss-dashcreate] and [Dashboard widgets — bubble/scatter][ss-bubble], [Editable drill-in grids][ss-drillin]:

| SmartSuite widget category                   | Examples                                                                                              | FLYTEHAUS state                                                                                                    | Severity                                               |
| -------------------------------------------- | ----------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------ |
| **Page Display**                             | Hero, Banner, Heading, FAQ, Webpage, Text, Spacing, Divider                                           | None — `console/dashboards/page.tsx` is hardcoded sections                                                         | P2                                                     |
| **Record Display**                           | Record Selector, Record Details, Record History, Record Comments, My Work                             | Partial — `console/page.tsx` shows recent projects table                                                           | P2                                                     |
| **Action**                                   | Filter Widget, Button Row, Filter Banner, "Create a Record" button ([button action][ss-buttonaction]) | None                                                                                                               | P1                                                     |
| **Metrics**                                  | Comparison Metric, Progress Metric, Summary Card                                                      | `MetricCard` exists with optional `Sparkline` (`src/components/ui/MetricCard.tsx`) — **single-purpose, hardcoded** | P1                                                     |
| **Charts**                                   | Bar, Column, Line, Area, Pie, Donut, Scatter, Bubble, Heatmap, Combo + drill-in editable grid         | recharts is in deps, used in 2 pages — **no widget-style chart authoring**                                         | **P0** — biggest dashboard gap.                        |
| **Views (embedded grid)**                    | Embed any saved view inside a dashboard                                                               | None                                                                                                               | P1                                                     |
| **Integration / Utility**                    | World clock, countdowns, Vimeo, YouTube, Google Docs/Meet/Sheets, Figma                               | None                                                                                                               | P3 — luxury, not core.                                 |
| **Dashboard layout**                         | Drag-drop grid, navigation tabs, page width modes, footer ([Dashboard Settings][ss-dashsettings])     | Static React tree                                                                                                  | P1                                                     |
| **Public sharing**                           | Unique link, passcode, regenerate, embed code, hide toolbar, read-only ([Public Sharing][ss-pubdash]) | None                                                                                                               | **P0** — clients ask for "send me a dashboard" weekly. |
| **PDF export**                               | Yes, with format/layout/font controls ([PDF export][ss-pdfdash])                                      | None (we have `@react-pdf/renderer` for proposals/guides — different code path)                                    | P1                                                     |
| **Drill-in to underlying records, editable** | Yes ([Editable Drill-in Grids][ss-drillin])                                                           | DataTable rows link to detail pages, but no modal drill-in from a chart                                            | P1                                                     |

**Recommended chart library:** `recharts` (already installed `^3.8.1`). It covers bar/column/line/area/pie/donut/scatter/composed natively. For heatmap and bubble, recharts can be coaxed (Scatter has a `ZAxis` for bubble); a `<HeatmapGrid>` SVG component is ~100 LOC. Avoid adding a second chart dep; the `ChartShell` wrapper (`src/components/charts/ChartShell.tsx`) already standardises loading/error/empty.

## 6. Top 10 implementation recommendations

Ranked by impact ÷ effort. Effort: **S** ≤ 1 day · **M** ≤ 1 week · **L** ≤ 2 weeks · **XL** > 2 weeks.

| #   | Recommendation                                                    | Why it wins                                                                                                                                                                                                                                                              | Files / approach                                                                                                                                                                                            | Effort                         |
| --- | ----------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----- |
| 1   | **Promote `SavedView` to a sharable, named `view_configs` table** | Unlocks every other view-type below — once views are addressable, swapping the renderer is trivial. Today views are user-scoped JSON; lift to `view_configs(id, org_id, table_id, name, type, config jsonb, scope: private                                               | org                                                                                                                                                                                                         | public)` with RLS.             | New migration + `src/lib/db/views.ts`; rewrite save path in `DataTableInteractive.tsx#L132` to write to the new table when `scope !== 'private'`; sidebar selector reads list. | **L** |
| 2   | **Generic `<KanbanBoard>` component**                             | Every status-bearing table immediately gets a view with zero per-page work. Re-uses `@dnd-kit` already in `DataTableInteractive`. Status enums already exist on `tickets`, `tasks`, `punch_items`, `rfis`, `submittals`, `requisitions`, `purchase_orders`, `incidents`. | New `src/components/views/KanbanBoard.tsx` taking `{ rows, laneField, cardFields[], onMove }`. Wire into `console/tasks`, `console/punch`, `console/rfis` first.                                            | **M**                          |
| 3   | **Spotlight / conditional formatting on `DataTable`**             | One feature unlocks 30 visual signals (overdue red, complete green, blocked amber). Operators stop having to scan.                                                                                                                                                       | Extend `Column<T>` with `rowClassName?: (row) => string` and add a `spotlight: { rules: [{ when, color, scope }] }` prop on `DataTable` saved-view config. CSS-token only — no runtime cost.                | **S**                          |
| 4   | **Generic `<CalendarView>` (day/week/month/agenda)**              | Replace bespoke `ScheduleCalendar.tsx` with a primitive every date-bearing table can consume. Drag-to-reschedule turns it from read-only into a working surface.                                                                                                         | New `src/components/views/CalendarView.tsx`; reuse FullCalendar-free homegrown grid (we already have month/week math in `ScheduleCalendar`). Wire to `events`, `tasks`, `inspections`, `safety_briefings`.  | **L**                          |
| 5   | **Column totals / summary footer**                                | Finance and budget tables (`console/finance/budgets`, `pay-apps`, `expenses`, `invoices`) are practically unusable without footer sums. Already have `tabular: true` on numeric cols in the codebase.                                                                    | Add `total?: 'sum'\|'avg'\|'min'\|'max'\|'count'` to `Column<T>`, render `<tfoot>` in `DataTableInteractive`.                                                                                               | **S**                          |
| 6   | **Generic `<ChartView>` config-driven on top of `recharts`**      | Stops the copy-paste between `ReportsCharts`, `CarbonCharts`, future modules. Lets ops reshape a chart without a deploy.                                                                                                                                                 | New `src/components/views/ChartView.tsx` taking `{ type: 'bar'\|'line'\|'pie'\|..., data, xKey, yKey, groupBy, agg }`. Backed by `ChartShell`.                                                              | **M**                          |
| 7   | **Composable Dashboard canvas**                                   | Replaces the hardcoded `console/dashboards/page.tsx`. Even a minimum-viable widget palette (4 widget types: KPI / chart / saved-view-embed / markdown) closes most of the SmartSuite gap.                                                                                | New `src/app/(platform)/console/dashboards/[id]/page.tsx` reading a `dashboards(id, layout jsonb, widgets jsonb[])` row; render `Widget` components in a CSS grid. Use existing `@dnd-kit` for layout edit. | **XL** — but the MVP is **L**. |
| 8   | **Public dashboard / saved-view share link**                      | Every client meeting ends with "can you send me this view?" Today the answer is screenshot. Tomorrow: `/p/[slug]/dashboard/[shareToken]` with HMAC-signed token in `urlFor("portal", ...)`.                                                                              | New `share_tokens(token, view_id, org_id, expires_at, passcode_hash)` table; portal route `/p/[slug]/v/[token]`; reuse `GVTEWAY` shell.                                                                     | **M**                          |
| 9   | **`<TimelineView>` (zoomable swimlane bars)**                     | Bridges to Gantt without committing to dependency math. Covers `events`, `rentals`, `transport`, the 8-Phase Lifecycle.                                                                                                                                                  | New `src/components/views/TimelineView.tsx` — d3-scale or hand-rolled SVG; lanes on Y by `groupBy`, bars on X by `dateRange`.                                                                               | **L**                          |
| 10  | **Inline cell edit on `DataTable`**                               | Closes the perceived gap to a spreadsheet. Fill handle + bulk paste are nice; inline edit is the keystone.                                                                                                                                                               | Extend `Column<T>` with `editable?: { kind: 'text'\|'select'\|'date', mutate: (id, value) => Promise }`; render `<input>` on focus; persist via existing server actions.                                    | **L**                          |

Honourable mentions (P2/P3): generic `<MapView>` (`maplibre-gl` already in deps; lift `MapShell` to consume any address column), filter operators (`>`, `<`, `contains`, `between`), undo/redo bus, locked + foldered saved views, public Form View (connect existing `form_defs` to `(portal)` shell).
