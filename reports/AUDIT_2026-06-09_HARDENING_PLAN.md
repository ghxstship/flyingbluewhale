## Decisions (resolved 2026-06-09, user-approved)

1. **Plan scope:** APPROVED — all 8 phases, executed in order, one validated commit batch per phase to main.
2. **Marketplace apply/review flows (3.4):** BUILD — implement `job_applications` / `open_call_submissions` / `reviews` write paths (apply pages, /me views, review forms).
3. **Arabic locale (5.6):** FINISH `ar.json` — translate the ~13.4k missing keys (same batch approach as es/fr/de/pt/ja).
4. **`status` columns (6.3):** FULL RENAME — no legacy references allowed. All 75 bare `status` columns migrate to `*_phase`/`*_state` names plus every TypeScript reference. The Phase-0 canon-test allowlist entry for the baseline is temporary and is removed when the rename lands.
5. **Support email (5.3):** defaulting to `sos@atlvs.pro` — `src/lib/brand.ts` is the brand SSOT per canon; contact/help pages align to it. (Flag if wrong.)
6. **`incidents` soft delete (2.2):** add `deleted_at` column via migration (consistent with the other 74 soft-deletable tables) rather than shrinking the allowlist.
